如何利用 Github Page 和 Hexo 搭建个人博客

置顶 | 发表于 2022-10-12 更新于 2022-10-13 阅读次数：
本文字数： 1.8k 阅读时长 ≈ 2 分钟

哪位理工男不想要一个属于自己的网站呢来记录学习心得装13呢？哪个男朋友不想要一个自己的网站记录与女朋友的点滴撒狗粮呢？且北邮双创实践课要求搭建一个自己的博客且如果追求分数 90+ 需要部署到互联网，考虑到目前的其他教程大多较旧，本人搭建过程也遇到了很多坑，固有本文

个人搭建博客分为动态博客和静态博客两种，动态博客强制需要服务器或者虚拟主机，在主机控制台一键安装 Typecho 或者 WordPress，博客便构建完成，动态博客能实现随时写随时发布，甚至在手机上编辑博文，类似个人构建的单人微博，方便快捷，但缺点是成本较高，除了域名还需要额外考虑服务器成本
本文选择了 Github Page 和 Hexo 搭建的静态博客，静态博客写完以后需要手动执行一些命令将文章转换成最终的网页文件，该操作一般只能在电脑上执行，且许多功能如评论需要自己额外添加，但利用 Github Page 免去了服务器成本，另外，自己动手自定义而不是一键傻瓜式操作是让程序员动心的一件事不是吗找罪受？

安装 Git

在官网下载并安装，现在一般都是 64 位系统，选择 Setup 版本而不是 Portable 便携版本
安装过程中注意以下事项，其他保持默认

  • 更改安装目录，别 C 盘就行
  • 选择 Git from the command line and also from 3rd-party software
  • 选择 Enable file system caching 和 Enable Git Credential Manager

Git 绑定 Github

虽然在 GitHub 上我们可以直接上传文件，但是并不好用，所以我们利用 SSH 来完成 GitHub 的绑定并提交文件

SSH（安全外壳协议，Secure Shell 的缩写）是建立在应用层基础上的安全协议。SSH 是目前较可靠，专为远程登录会话和其他网络服务提供安全性的协议，利用 SSH 协议可以有效防止远程管理过程中的信息泄露问题。简单来说，SSH 能保障你的账户安全，将你的数据加密压缩，不仅防止其他人截获你的数据，还能加快传输速度\

要用 git 上传文件到 GitHub 首先得利用 SSH 登录远程主机，这里使用公钥登录\

在 Git Bash 中输入命令 ssh-keygen -t rsa,指定 RSA 算法生成密钥，然后敲四次回车键，之后就就会生成两个文件，分别为秘钥 id_rsa 和公钥 id_rsa.pub

值得注意的是，git 中的复制粘贴不是 Ctrl+C 和 Ctrl+V，而是 Ctrl+insert 和 Shift+insert
复制从 ssh-rsa 开始（包含）的后面所有内容
在 Github 主页，点击右上角向下箭头，选择 settings，选择 SSH and GPG keys，再点击 New SSH key
将复制的内容粘贴到 Key 内，Title 不用填，Add SSH key 确认即可
再在 Git Bash 中输入 ssh -T git@github.com，根据显示，输入 yes 即可
